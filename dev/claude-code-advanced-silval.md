## 📋 영상 구조 (43분)

### 1. Skills — AI에게 주는 업무 매뉴얼 (2:04)

**개념:** 재사용 가능한 업무 자동화 시스템
- 반복되는 작업 패턴을 스킬 파일로 정의
- Claude Code가 스킬을 참조해서 일관된 작업 수행
- 예: "컴포넌트 생성 스킬", "API 엔드포인트 스킬", "테스트 작성 스킬"

**핵심:** 매번 프롬프트를 다시 쓰지 않아도, 스킬 파일 하나로 표준화된 결과물 생성

### 2. Sub-Agent & 병렬 실행 (16:18)

**개념:** 독립 컨텍스트에서 여러 작업을 동시 실행
- 메인 에이전트가 작업을 분배 → 서브에이전트가 병렬 처리
- 각 서브에이전트는 **독립된 컨텍스트** (서로 간섭 없음)
- **Git Worktree**와 결합하면 충돌 없는 병렬 코딩 가능

**실전 예시:**
```
메인: "이 10개 피드백을 각각 수정해줘"
  → Sub-Agent 1: 피드백 #1 수정
  → Sub-Agent 2: 피드백 #2 수정
  → ...
  → Sub-Agent 10: 피드백 #10 수정
→ 전부 완료 후 머지
```

**Git Worktree:**
- 하나의 리포에서 여러 브랜치를 동시에 체크아웃
- 서브에이전트마다 독립된 작업 디렉토리
- 병렬 작업해도 파일 충돌 없음

### 3. Hooks — 자동화 엔진 (26:48)

**개념:** 이벤트 기반 자동화
- 특정 이벤트 발생 시 자동으로 스크립트 실행
- 예: 커밋 전 자동 lint, 파일 저장 시 자동 테스트

**활용:**
- PreCommit Hook: 커밋 전 자동 코드 검증
- PostSave Hook: 파일 변경 시 자동 빌드
- 커스텀 Hook: 원하는 이벤트에 원하는 동작 연결

### 4. 고급 팁 & 생산성 극대화 (30:29)

**음성 입력:**
- 복잡한 프롬프트를 타이핑 대신 **말로 전달**
- 긴 설명이 필요한 작업에 특히 유용

**커스텀 MCP 서버:**
- 경쟁 모델(GPT 등)로 Plan Review 가능
- Claude가 만든 계획을 다른 AI가 검증하는 구조
- 다중 AI 크로스체크

### 5. 통합 데모 — 피드백 10개 한큐 수정 (36:43)

**실전 파이프라인:**
```
1. 클라이언트 피드백 10개 리스트
2. Skills로 수정 패턴 정의
3. Sub-Agent 10개 병렬 실행 (각 피드백별)
4. Git Worktree로 충돌 방지
5. Hooks로 자동 검증
6. 전부 머지 → 완료
```

한 사람이 10개 피드백을 **동시에** 처리 = 팀급 생산성

## 💡 핵심 인사이트

1. **Skills = 재사용 가능한 프롬프트 라이브러리** — 반복 작업의 품질 일관성 보장
2. **Sub-Agent + Git Worktree = 1인 팀** — 병렬 실행이 진짜 게임 체인저
3. **Hooks = 자동화 접착제** — 수동 검증을 자동화로 전환
4. **음성 입력 = 프롬프트 병목 해소** — 복잡한 요구사항은 말로
5. **MCP로 다중 AI 크로스체크** — Claude 계획을 GPT가 리뷰
